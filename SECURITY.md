# Security Policy

Please report security vulnerabilities **privately** through GitHub
Security Advisories — use the "Report a vulnerability" button under
the [Security tab](../../security/advisories/new).

Do not file a public issue or open a PR for a security concern — that
would expose the issue before we can release a fix.

We aim to acknowledge reports within 72 hours and to provide a
remediation timeline within one week.

## Scope

In scope:
- Code in this repository (kutl OSS — sync engine, CLI, daemon, relay).
- The released CLI/daemon binaries published under
  `github.com/kutl-io/kutl/releases`.

Out of scope (please report directly to the relevant operator):
- Third-party dependencies — please report upstream first.
- Self-hosted deployments configured by third parties.

We do not currently run a paid bug bounty program. Substantive reports
will be credited (with your consent) in the relevant release notes.
